### 上下游
> A 服务 --> B 服务；像河流一下，箭头方向表示水流向。

- 上游：调用方
- 下游：被调用方


### 透明与不透明
> 对于【调用者】来说，是否能看得见【被调用者】内部细节

- 透明：调用者可以看到内部结构、字段、实现细节。

```go
type User struct {
    Name string
    Age  int
}

// 外部可以直接修改内部属性：
u.Name = "Tom"
```

- 不透明：调用者只能使用接口，看不到内部结构。

```go
type User struct {
    name string
    age  int
}

// 外部只能通过接口：
name := u.getName()
```
